# GMP Validation Posture

DPI TRS Tracker is open-source decision-support software for pharmaceutical manufacturing performance tracking.

It is not validated GMP software by default.

## What this project can do

- Help teams explore TRS/OEE workflows
- Provide transparent calculation logic
- Support learning, prototyping, and open-source collaboration
- Demonstrate production KPI and batch-follow-up concepts
- Generate draft operational summaries when AI features are added

## What this project must not claim by default

- It must not claim to replace approved batch records.
- It must not claim to make batch release decisions.
- It must not claim validated GMP status without a formal validation package.
- It must not store real confidential production, supplier, employee, patient, or batch-sensitive data in the public repository.

## Requirements before regulated use

A site considering regulated use would need, at minimum:

- Approved user requirements
- Risk assessment
- Validation plan and validation report
- Controlled configuration and change control
- Access control and role review
- Audit trail assessment
- Data backup and retention rules
- ALCOA+ data-integrity review
- QA approval
- Training and SOP updates

## AI-generated content

Any AI-generated summary, investigation prompt, release note, deviation prompt, or production review text must be treated as draft decision support. It requires human review by authorized personnel before operational or quality use.

## Public demo data rule

Use only fictional demo data in the public repository. Do not commit real site names, product names, batch numbers, employee names, supplier information, patient data, deviations, complaints, or confidential production metrics.
