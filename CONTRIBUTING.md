# Contributing

Thank you for considering a contribution to DPI TRS Tracker.

This project is an open-source pharmaceutical manufacturing performance tracker focused on TRS/OEE, batch-level visibility, and production follow-up.

## Who can contribute

- Production managers
- Pharmacists and QA partners
- Industrial engineers
- Developers interested in manufacturing systems
- Students learning open-source industrial software

## Useful contributions

- Improve TRS/OEE calculation documentation
- Add tests for calculation logic
- Improve frontend workflows for operators and supervisors
- Improve API validation and error handling
- Add safe fictional demo data
- Improve deployment documentation
- Review GMP/data-integrity assumptions

## GMP and validation note

This project is open-source decision-support software. It is not validated GMP software by default. Any regulated use requires formal validation, QA approval, access control, audit trail review, and site-specific procedures.

## Pull request checklist

- Explain the operational problem solved.
- Keep calculations transparent.
- Do not include confidential production, employee, supplier, patient, or real batch data.
- Update documentation when behavior changes.
- Add or update tests for calculation changes.
