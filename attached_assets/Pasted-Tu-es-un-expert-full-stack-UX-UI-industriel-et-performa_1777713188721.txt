Tu es un expert full-stack, UX/UI industriel et performance pharmaceutique GMP.

Construis une application web responsive + mobile-first pour le suivi TRS/OEE d’une unité DPI pharmaceutique, avec deux lignes principales :
1. Géluleuse Harro Höfliger / Modu-C
2. Blistereuse IMA TR135 S

Contexte métier :
L’application doit suivre la performance quotidienne, mensuelle et annuelle des équipements DPI selon la logique NF E 60-182 / TRS / TRG / TRE.

Objectif :
Créer une application simple, robuste et exploitable en production avec 3 vues :
- Vue Opérateur
- Vue Superviseur
- Vue Admin

Stack souhaitée :
- Frontend : React + TypeScript
- UI : Tailwind CSS
- Backend : Node.js + Express
- Database : PostgreSQL ou Supabase
- Authentification par rôles
- Application responsive mobile/tablette/desktop
- Prévoir PWA pour utilisation mobile

Indicateurs à calculer :
- tT : temps total calendrier
- tO : temps d’ouverture
- tR : temps requis
- tF : temps de fonctionnement
- tN : temps net
- tU : temps utile
- DO : disponibilité opérationnelle
- TP : taux de performance
- TQ : taux qualité
- TRS = DO × TP × TQ
- TRG
- TRE
- Cadence théorique
- Cadence validée produit
- Écart cadence
- Quantité produite
- Quantité conforme
- Rebuts
- Arrêts planifiés
- Arrêts non planifiés

Règle importante :
Le TRS mensuel ne doit pas être une simple moyenne des TRS journaliers. Il doit être calculé sur les totaux mensuels :
TRS mensuel = Somme temps utile / Somme temps requis
ou équivalent DO × TP × TQ sur les totaux consolidés.

Classification des arrêts :
Créer un dictionnaire paramétrable des arrêts avec impact temps :
- Pause : impact TO
- CHSG : changement série / arrêt planifié / impact TR
- NET : nettoyage local et équipement / arrêt planifié / déduit du TR selon règles
- VA : vide d’atelier / arrêt planifié
- MQCH : manque charge / déduit du temps d’ouverture
- AG : arrêt géluleuse / arrêt non planifié / impact tF
- AB : arrêt blistereuse / arrêt non planifié / impact tF
- AI : arrêt intervention ou incident
- IM : intervention maintenance
- NQ : non-qualité / impact tU ou TQ
- UTL : utilités et environnement
- ORG : organisation
- PERF / SP : sous-performance / impact TP
- Écart cadence : différence entre cadence validée et cadence réelle

Règles de calcul :
Créer un moteur de calcul transparent avec formules visibles dans le code.

Formules de base :
tT = temps calendrier
tO = tT - fermetures / jours off / périodes hors ouverture
tR = tO - arrêts planifiés autorisés
tF = tR - arrêts non planifiés
tN = quantité produite / cadence de référence
tU = quantité conforme / cadence de référence

DO = tF / tR
TP = tN / tF
TQ = tU / tN
TRS = tU / tR
TRG = tU / tO
TRE = tU / tT

Prévoir gestion des cas :

- Jour off
- Valeurs négatives
- Arrêts qui se chevauchent
- Saisie incomplète
- Cadence manquante
- Quantité incohérente
