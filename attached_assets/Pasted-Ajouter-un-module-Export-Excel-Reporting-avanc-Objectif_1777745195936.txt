Ajouter un module “Export Excel Reporting” avancé.

Objectif :
L’application ne doit pas exporter uniquement des raw data.
Elle doit générer des fichiers Excel professionnels, fonctionnels et réutilisables hors application pour reporting production / direction.

Types d’exports Excel :
1. Rapport TRS journalier
2. Rapport TRS hebdomadaire
3. Rapport TRS mensuel
4. Rapport planning vs réalisé
5. Rapport arrêts / Pareto
6. Rapport responsable production DPI
7. Rapport complet mensuel direction

Exigences Excel :
- Fichier .xlsx natif
- Design professionnel
- Feuilles structurées
- Formules Excel visibles
- Tableaux filtrables
- Graphiques intégrés
- Mise en forme conditionnelle
- Cellules protégées pour formules
- Zones de saisie ou commentaire si nécessaire
- En-têtes propres avec logo société si disponible
- Pied de page avec date d’export, utilisateur, période, version

Le fichier exporté doit être utilisable indépendamment de l’application.

Structure du rapport mensuel :
Feuille 1 : Synthèse Direction
- KPI principaux :
  - TRS mensuel
  - TRG
  - TRE
  - DO
  - TP
  - TQ
  - Respect planning
  - Quantité planifiée
  - Quantité réalisée
  - Écart quantité
  - Nombre d’arrêts
  - Durée totale arrêts
- Cartes KPI avec couleurs :
  - Vert : conforme
  - Orange : vigilance
  - Rouge : critique
- Graphique TRS vs objectif
- Graphique respect planning
- Commentaire responsable production

Feuille 2 : Dashboard TRS
- Tableau par jour :
  - Date
  - Jour
  - Équipement
  - Produit
  - Lot
  - Quantité planifiée
  - Quantité produite
  - Quantité conforme
  - Rebuts
  - tT
  - tO
  - tR
  - tF
  - tN
  - tU
  - DO
  - TP
  - TQ
  - TRS
  - TRG
  - TRE
  - Statut
- Formules Excel réelles :
  - DO = tF / tR
  - TP = tN / tF
  - TQ = tU / tN
  - TRS = tU / tR
  - TRG = tU / tO
  - TRE = tU / tT
- Ne pas remplacer les formules par des valeurs figées sauf si option choisie.

Feuille 3 : Planning vs Réalisé
- Date
- Activité
- Local
- Équipement
- Produit
- Lot
- Quantité planifiée
- Quantité réalisée
- Écart
- Taux respect planning
- Statut activité
- Commentaire
- Formule :
  - Écart = Réalisé - Planifié
  - Respect planning = Réalisé / Planifié
- Graphique avancement par activité

Feuille 4 : Arrêts
- Date
- Équipement
- Local
- Produit
- Lot
- Type arrêt
- Planifié / Non planifié
- Début
- Fin
- Durée
- Impact KPI
- Commentaire
- Responsable action
- Statut
- Mise en forme conditionnelle selon criticité

Feuille 5 : Pareto Arrêts
- Type arrêt
- Nombre d’occurrences
- Durée totale
- % contribution
- % cumulé
- Graphique Pareto intégré
- Formules Excel pour % et cumul

Feuille 6 : Statut Équipements & Locaux
- Équipement / Local
- Statut
- Activité en cours
- Produit
- Lot
- Avancement
- Dernier événement
- Durée état actuel
- Problème actif
- Commentaire

Feuille 7 : Données Sources

Feuille 8 : Paramètres
- Objectifs TRS
- Seuils couleurs

Design Excel :
- Thème sobre industriel
- Bleu foncé / gris / blanc
- Vert pour conforme
- Orange pour vigilance
- Rouge pour critique
- Gris pour jour off ou non applicable
- Titres clairs
- Largeurs colonnes ajustées
- Figer les volets
- Filtres activés
- Format pourcentages propre
- Format heures/durées lisible
- Protection des cellules de formules

Fonctionnalités :
- Bouton dans l’app : “Exporter rapport Excel”
- Choix de période :
  - jour
  - semaine
  - mois
  - année
  - plage personnalisée
- Choix équipement :
  - tous
  - géluleuse
  - blistereuse
  - ligne secondaire
- Choix format :
  - rapport complet
  - rapport direction
  - rapport technique détaillé
  - raw data uniquement
- Option :
  - Export avec formules
  - Export valeurs figées
  - Export avec feuilles protégées
  - Export avec données sources visibles ou masquées

Librairie recommandée :
Utiliser ExcelJS côté backend pour générer les fichiers .xlsx avec :
- styles
- formules
- tableaux
- graphiques si possible
- filtres
- protection feuilles
- mise en forme conditionnelle

Si ExcelJS ne supporte pas bien les graphiques :
- Générer au minimum les tableaux propres avec formules
- Créer les zones de données prêtes pour graphique
- Documenter la limite
- Prévoir une version future avec template Excel préformaté

Approche recommandée :
Créer des templates Excel :
- templates/report_monthly.xlsx
- templates/report_weekly.xlsx
- templates/report_direction.xlsx

L’application remplit les templates avec les données exportées.
Cela permet d’avoir un design stable, joli et facilement modifiable hors code.

Créer un service backend :
excelReportService.ts

Fonctions :
- generateDailyReport()
- generateWeeklyReport()
- generateMonthlyReport()
- generatePlanningVsActualReport()
- generateDowntimeParetoReport()
- generateDirectionReport()

Chaque rapport doit :
- récupérer les données depuis Supabase
- calculer les agrégats nécessaires
- écrire les formules Excel
- appliquer le design
- retourner un fichier .xlsx téléchargeable

Exigence importante :
Le fichier Excel généré doit pouvoir être envoyé par mail à la direction et rester compréhensible sans ouvrir l’application.
