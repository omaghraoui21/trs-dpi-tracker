Améliore l’application TRS DPI en ajoutant un module d’ingestion du planning hebdomadaire de production Excel sous le format fourni.

Format planning à supporter :
- Planning hebdomadaire type “PLANNING HEBDOMADAIRE DE PRODUCTION SITE EL FEJJA”
- Semaine en haut : S17 2026, S18 2026, etc.
- Colonnes par jour : Lundi, Mardi, Mercredi, Jeudi, Vendredi, Samedi, Dimanche
- Lignes par activité :
  - Pesée
  - Fabrication
  - Mise en gélules
  - Conditionnement primaire
  - Conditionnement secondaire & tertiaire 1ère ligne
  - Conditionnement secondaire & tertiaire 2ème ligne
- Pour chaque activité :
  - Produit
  - Lot
  - Quantité planifiée si présente dans la cellule, exemple : 360 000 UN, 6 000 UN, 4 000 UN
  - Type d’activité spéciale si présente : Entretien préventif, Nettoyage, Étalonnage

Créer un module “Import Planning” :
- Upload fichier Excel
- Lecture automatique de la semaine
- Lecture automatique des jours
- Transformation du planning en données structurées
- Prévisualisation avant import
- Validation par responsable production
- Détection des anomalies :
  - produit sans lot
  - lot sans produit
  - quantité non détectée
  - activité vide
  - doublon lot / jour / activité
  - activité maintenance ou nettoyage détectée comme non-production

Ajouter KPI respect planning :
- Quantité planifiée
- Quantité réalisée
- Écart quantité = réalisée - planifiée
- Taux respect planning = quantité réalisée / quantité planifiée
- Taux avancement activité = réalisé / planifié
- Nombre de lots planifiés
- Nombre de lots réalisés
- Nombre de boîtes planifiées
- Nombre de boîtes fabriquées
- Nombre de gélules planifiées
- Nombre de gélules réalisées
- Retard planning
- Activités non démarrées
- Activités en cours
- Activités terminées
- Activités bloquées

Règles KPI :
- Pour géluleuse : comparer gélules réalisées vs gélules planifiées.
- Pour blistereuse / conditionnement : comparer boîtes ou blisters réalisés vs planifiés selon unité disponible.
- Si unité = UN, permettre à l’admin de définir si UN signifie gélules, blisters ou boîtes selon activité.
- Taux respect planning = réalisé / planifié × 100.
- Si planifié = 0, ne pas calculer et afficher “N/A”.
- Si réalisé < 90 % du planifié : statut vigilance.
- Si réalisé < 75 % du planifié : statut critique.
- Si arrêt machine actif : statut problème.

5. Notifications
Créer un système de notifications :
- Panne équipement
- Arrêt supérieur à 15 min
- Retard planning supérieur à 30 min
- Quantité réalisée inférieure à 75 % du plan
- TRS inférieur objectif
- TQ inférieur seuil
- Activité planifiée non démarrée
- Lot sans saisie opérateur
- Incohérence planning vs production
- Local bloqué ou non disponible

- Badge rouge dans dashboard

- Possibilité d’accuser réception

Vue opérateur améliorée :
- Après import planning, l’opérateur voit uniquement les activités prévues pour son équipement et son jour.
- Il peut cliquer “Démarrer activité”.
- Il saisit quantité réalisée progressivement.
- Il déclare arrêt ou panne.
- Il clôture activité.
- L’app calcule automatiquement avancement planning.

Vue superviseur améliorée :
- Voir toutes les activités du jour.
- Réaffecter activité à une autre équipe.
- Corriger produit / lot si erreur planning.
- Valider les quantités réalisées.
- Confirmer les arrêts.
- Ajouter commentaire de justification.

Vue admin améliorée :
- Paramétrer mapping activité → équipement / local.
- Paramétrer unités : gélules, blisters, boîtes, kg, lots.
- Paramétrer seuils d’alerte.
- Paramétrer objectifs TRS et respect planning.
- Paramétrer délais tolérés avant notification.
