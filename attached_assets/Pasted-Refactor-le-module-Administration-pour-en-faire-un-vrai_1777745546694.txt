Exigence critique :
Le module Admin doit permettre de reprogrammer le comportement métier de l’application sans modifier le code source.

Principe :
La logique métier doit être data-driven :
- formules depuis calculation_formulas
- arrêts depuis downtime_categories
- affichages depuis dashboard_display_settings
- objectifs depuis kpi_targets
- cadences depuis product_equipment_cadences
- mappings depuis planning_activity_mappings
- alertes depuis notification_rules

Livrables :
- pages Admin complètes
- tables SQL
- seed data
- services backend
- validation des formulaires
- audit trail
- README expliquant comment modifier les règles de calcul et affichage
